1. TEXNIK STACK VA KUTUBXONALAR (TECHNICAL STACK & LIBRARIES)
--------------------------------------------------------------------------------
Tizim to'liq asinxron va samarali ishlashi uchun quyidagi texnologiyalar tanlandi:

* Dasturlash tili: Python 3.11 / 3.12 (Asinxron ishlash va barqarorlik uchun)
* Asosiy Bot kutubxonasi: aiogram v3.x (Admin va Taksilar uchun asosiy boshqaruv boti)
* Userbot kutubxonasi: Pyrogram v2.0+ (Taksi hisobiga kirib, guruhlarga xabar yuborish uchun)
* Ma'lumotlar bazasi: Supabase (PostgreSQL) - Bepul, uzoq muddatli va juda tezkor.
* Web-ramka (Admin Panel): FastAPI + Jinja2 Templates (Render.com da bitta port orqali ham API, ham veb-sahifani boshqarish uchun juda qulay)
* Vazifalar rejalashtiruvchisi (Scheduler): APScheduler (Asinxron tarzda xabarlarni kerakli vaqtda yuborib turish uchun)
* Konteynerizatsiya: Docker va Docker Compose (Render.com ga oson deploy qilish va lokal testlar uchun)

2. SUPABASE MA'LUMOTLAR BAZASI TUZILISHI (DATABASE SCHEMA)
--------------------------------------------------------------------------------
Render serverlari o'chib yonganida yoki uyqu rejimiga o'tganida barcha sessiyalar va
ma'lumotlar saqlanib qolishi uchun barcha holatlar Supabase (PostgreSQL) da saqlanadi.

Quyida bazadagi asosiy jadvallar va ularning SQL ko'rinishi keltirilgan:

-- 1. Foydalanuvchilar (Taksilar va Adminlar) jadvali
CREATE TABLE users (
    telegram_id BIGINT PRIMARY KEY,
    username VARCHAR(100),
    first_name VARCHAR(100),
    phone_number VARCHAR(20),
    language VARCHAR(5) DEFAULT 'uz_lat', -- uz_lat, uz_cyr
    session_string TEXT, -- Pyrogram String Session (shifrlangan yoki ochiq holda)
    status VARCHAR(30) DEFAULT 'new', -- new, pending_code, pending_2fa, active, blocked
    trial_messages_left INT DEFAULT 15, -- Bepul sinov tarifidagi xabarlar soni
    subscription_end TIMESTAMP WITH TIME ZONE NULL, -- Obuna tugash vaqti
    blocked_until TIMESTAMP WITH TIME ZONE NULL, -- Umuman bloklanganlik muddati
    msg_blocked_until TIMESTAMP WITH TIME ZONE NULL, -- Vaqtinchalik xabar yuborish bloklanganlik muddati
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 2. Taksilarning guruhlari jadvali
CREATE TABLE user_groups (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_telegram_id BIGINT REFERENCES users(telegram_id) ON DELETE CASCADE,
    chat_id BIGINT NOT NULL,
    group_title VARCHAR(255),
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 3. Taksilarning reklama xabarlari va ularning sozlamalari
CREATE TABLE user_messages (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_telegram_id BIGINT REFERENCES users(telegram_id) ON DELETE CASCADE,
    message_text TEXT,
    interval_minutes INT DEFAULT 2, -- Minimal vaqt 2 daqiqa
    is_running BOOLEAN DEFAULT FALSE,
    duration_hours INT DEFAULT 0, -- 0 bo'lsa cheksiz (foydalanuvchi o'zi to'xtatguncha)
    started_running_at TIMESTAMP WITH TIME ZONE NULL,
    last_sent_at TIMESTAMP WITH TIME ZONE NULL
);

-- 4. Tizim xavfsizligi va xabarlar statistikasi
CREATE TABLE statistics (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_telegram_id BIGINT REFERENCES users(telegram_id) ON DELETE CASCADE,
    sent_count INT DEFAULT 0,
    failed_count INT DEFAULT 0,
    last_updated TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 5. Admin hisoblari (Veb panelga kirish uchun)
CREATE TABLE system_admins (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(20) DEFAULT 'admin',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

3. TELEGRAM SESSIYALARINI SUPABASE'DA XAVFSIZ SAQLASH (SESSION PERSISTENCE)
--------------------------------------------------------------------------------
Muammo: Render'ning bepul tarifi har 15 daqiqa faolsizlikdan so'ng serverni o'chiradi (sleep) va har deploy bo'lganda fayllarni to'liq o'chirib tashlaydi. SQLite fayllari yo'qolib ketadi va taksilar qaytadan tizimga kirishi kerak bo'ladi.

Yechim (String Session orqali saqlash):
1. Pyrogram kutubxonasida session fayli o'rniga "String Session" (belgilar ketma-ketligi ko'rinishidagi sessiya) dan foydalanamiz.
2. Taksi tizimga muvaffaqiyatli kirganda, uning hisobidan `await client.export_session_string()` funksiyasi orqali sessiya kodi olinadi.
3. Ushbu sessiya kodi Supabase'dagi `users` jadvalining `session_string` ustuniga saqlab qo'yiladi.
4. Tizim qayta yuklanganda yoki xabar yuborish vaqti kelganda:
   - Bazadan `session_string` yuklanadi.
   - Pyrogram Client obyekti ushbu string yordamida tezkor ravishda xotirada (RAM) ishga tushiriladi:
     `client = Client(name="user_session", session_string=session_string, api_id=API_ID, api_hash=API_HASH)`
   - Xabar guruhlarga muvaffaqiyatli yuborilgach, client to'xtatiladi yoki xotirada saqlab turiladi.
5. Hech qanday `.session` fayli diskda saqlanmaydi, natijada Render serveri o'chib yonsa ham sessiya yo'qolmaydi!

4. INLINE KLAVIATURA ORQALI XAVFSIZ KIRISH TIZIMI (SECURE INLINE LOGIN FLOW)
--------------------------------------------------------------------------------
Telegram akkauntiga bot ichida kirish foydalanuvchi uchun juda qulay va xavfsiz bo'lishi lozim.

Bosqichma-bosqich kirish oqimi:
1. Foydalanuvchi botda "Akkauntni ulash" (Kirish) tugmasini bosadi.
2. Bot undan telefon raqamini kiritishni so'raydi (masalan: `+998941080916`).
3. Bot asinxron tarzda vaqtincha Pyrogram client yaratib, Telegram'ga kod so'rovini yuboradi:
   `phone_code_hash = await client.send_code_request(phone_number)`
4. Telefon raqami va `phone_code_hash` ma'lumotlari bazada vaqtincha saqlanadi.
5. Bot foydalanuvchiga raqamli Inline klaviaturani taqdim etadi:
   ┌───────┬───────┬───────┐
   │   1   │   2   │   3   │
   ├───────┼───────┼───────┤
   │   4   │   5   │   6   │
   ├───────┼───────┼───────┤
   │   7   │   8   │   9   │
   ├───────┼───────┼───────┤
   │   ❌  │   0   │   ✅  │
   └───────┴───────┴───────┘
   Matn ko'rinishi: "Kodni kiriting: `_ _ _ _ _`"

6. Foydalanuvchi raqamlarni bosganda:
   - Bot har bir raqamni xotiradagi o'zgaruvchida (yoki vaqtinchalik DB keshda) saqlab boradi.
   - Ekrandagi matnni yulduzcha ko'rinishida yangilaydi (masalan, 3 ta raqam bosilsa: `* * * _ _`).
   - Bu xavfsizlik uchun juda muhim: Telegram chatda ochiq kod yozilganda kodni bekor qilishi mumkin. Yulduzchalar bu muammoni to'liq hal qiladi!
7. Foydalanuvchi `✅` tugmasini bosganida (yoki 5 ta raqam to'liq kiritilganda):
   - Kiritilgan kod tekshiriladi: `await client.sign_in(phone_number, phone_code_hash, code)`
   - Agar foydalanuvchida Ikki bosqichli parol (2FA) yoqilgan bo'lsa, `SessionPasswordNeeded` xatoligi yuzaga keladi. Bot bu holda asinxron tarzda 2FA parolini matn ko'rinishida yuborishni so'raydi.
   - Tizimga muvaffaqiyatli kirgach, `session_string` bazaga yoziladi va vaqtinchalik client tozalab tashlanadi. Foydalanuvchiga muvaffaqiyatli kirilganligi haqida xabar yuboriladi.

5. SPAMDAN HIMOYALANISH VA TELEGRAM CHEKLOVLARI (ANTI-SPAM & SAFETY LIMITS)
--------------------------------------------------------------------------------
Userbot orqali guruhlarga xabar yuborishda Telegram cheklovlariga (Limits) duch kelmaslik va hisob bloklanib (Spam) qolmasligining oldini olish choralari:

1. Vaqt oralig'i (Interval) nazorati:
   - Minimal xabar yuborish oralig'i qat'iy ravishda 2 daqiqa qilib belgilanadi.
   - Har bir xabar yuborishda tasodifiy kechikish (jitter) qo'shiladi (masalan, 2 daqiqaga qo'shimcha ravishda random holda 5-15 soniya). Bu Telegram algoritmlarini aldatishga yordam beradi va jarayonni inson faoliyatiga o'xshatadi.
2. Cheklovlar va FloodWait boshqaruvi:
   - Agar Telegram `FloodWait` (ko'p so'rov yuborish) xatoligini qaytarsa, kod buni avtomatik tutib oladi, yuborishni o'sha muddatga to'xtatib turadi (masalan, 300 soniya) va foydalanuvchiga ogohlantirish yuboradi.
3. Guruhlar soni cheklovi:
   - Bitta taksi foydalanuvchisi uchun bir vaqtning o'zida ko'pi bilan 10-15 ta guruhni tanlash imkoniyati beriladi. Juda ko'p guruhga xabar yuborish tezda bloklanishga olib keladi.
4. Xizmat shartlari (TOS) himoyasi:
   - Xabar matnida haqoratli yoki Telegram taqiqlagan so'zlar borligi tekshiriladi.

7. RENDER.COM FREE TIER OPTIMIZATSIYASI (KEEP-ALIVE MECHANISM)
--------------------------------------------------------------------------------
Render.com bepul xizmati 15 daqiqa davomida HTTP so'rov kelmasa, "uyqu" rejimiga o'tadi va barcha asinxron jarayonlarni to'xtatadi. Shuningdek, u faqat 1 ta ochiq portni (Web Port) taqdim etadi.

Yechim:
1. Yagona port arxitekturasi:
   - Docker konteynerida FastAPI loyihasi ishga tushiriladi (Render taqdim etgan `$PORT` da).
   - FastAPI ishga tushganida (startup event) background'da `aiogram` boti va `APScheduler` ishga tushirib yuboriladi. Hammasi bitta jarayon (process) va port ichida asinxron ishlaydi.
2. Keep-Alive (Uyg'oq saqlash tizimi):
   - FastAPI'da maxsus `/ping` endpointi yaratiladi.
   - Tashqi xizmatlar (masalan, UptimeRobot, Cron-Job.org yoki har 10 daqiqada ishlaydigan bepul ping-xizmat) orqali ushbu `/ping` endpointiga so'rov yuborib turiladi.
   - Bu Render serverini 24/7 uyg'oq saqlaydi va taksilarning reklamalari to'xtab qolmasligini kafolatlaydi.

9. BOSQICHMA-BOSQICH AMALGA OSHIRISH REJASI (ROADMAP)
--------------------------------------------------------------------------------

Bosqich 1: Loyihani sozlash va Ma'lumotlar bazasi (1-2 kun)
* GitHub repozitoriy ochish va loyiha papkalarini shakllantirish.
* Supabase'da loyiha yaratish va yuqoridagi SQL jadvallarini ishga tushirish.
* Ekologik o'zgaruvchilar (`.env`) sozlamalarini o'rnatish (API_ID, API_HASH, BOT_TOKEN, SUPABASE_URL, SUPABASE_KEY).

Bosqich 2: Asosiy Telegram Bot (aiogram) va Tizimga kirish (Userbot) (2-3 kun)
* Bot interfeysini ikki tilda (Lotin/Kirill) reply klaviatura bilan yaratish.
* Telefon raqami yordamida kirish va o'sha 5 xonali kodni yulduzcha ko'rinishidagi Inline tugmalar bilan qabul qilish logikasini yozish.
* Kod to'g'ri kiritilgach, `session_string` ni olib Supabase'ga muvaffaqiyatli saqlash.

Bosqich 3: Xabar yuborish logikasi va Guruhlarni sozlash (2 kun)
* Foydalanuvchining shaxsiy guruhlarini yuklash, guruhlarni tanlash tizimi.
* Reklama yozish va uning sozlamalarini (interval, davomiylik soati) bazaga yozish.
* APScheduler yoki asinxron `asyncio.sleep` tsikli yordamida guruhlarga avtomatik yuborishni boshlash. Har bir muvaffaqiyatli yoki muvaffaqiyatsiz xabar uchun statistikani yangilab borish hamda trial/obuna tekshiruvlarini amalga oshirish.

Bosqich 4: Admin Panel (FastAPI Webapp) (2-3 kun)
* FastAPI loyihasini sozlash va Jinja2 yordamida Admin boshqaruv paneli interfeysini tayyorlash.
* Dashboard, Taksilar ro'yxati, Foydalanuvchi kartasi (bloklash, obunani boshqarish) qismlarini ishlab chiqish.
* FastAPI bilan Telegram Botni bitta portda parallel ishga tushirish logikasini yozish.

Bosqich 5: Xavfsizlik, Testlash va Render.com deployment (2 kun)
* Kod xavfsizligini (FloodWait, Telegram spam-filtrlaridan qochish choralarini) sinash.
* Dockerfile va docker-compose.yml fayllarini tayyorlash.
* Render.com ga deploy qilish va Cron-Job/UptimeRobot orqali "Keep-Alive" (uyg'oq saqlash) tizimini ulash.
* Yakuniy topshirish va test qilish.
